# 会话列表UI更新

## 修改概述
已将左侧边栏的session下拉框改为**固定显示、可滚动的列表**，同时显示最多4个会话，超出部分可滚动查看。

## 修改的文件

### 1. `SessionDropdownSection.tsx` - 主要组件修改
- **移除下拉逻辑**：删除`isOpen`状态、下拉按钮点击、外部点击关闭逻辑
- **改为固定列表**：始终显示会话列表，不再需要点击展开
- **添加会话排序**：按活跃状态 → 消息数量 → 名称/ID 排序
- **更新渲染结构**：使用新的`sessionListContainer`和`sessionListItem`布局
- **保持功能**：点击切换、活跃状态指示器、消息计数显示

### 2. `SidebarPanel.module.css` - 样式更新
- **新增样式类**：
  - `.sessionListContainer`：固定高度容器（160px，约4个项目），带自定义滚动条
  - `.sessionListItem`：会话项样式，继承原下拉项的大部分样式
  - `.sessionItemContent`：内容布局容器
- **自定义滚动条**：暗色主题，与UI风格一致
- **保留现有样式**：`.sessionName`、`.sessionCount`、`.activeIndicator`继续使用

## 新UI特点

### 1. 始终可见的会话列表
- 不再需要点击按钮展开
- 会话列表直接显示在侧边栏中
- 当前选中会话有高亮背景（蓝色）

### 2. 固定高度设计
- 最大高度：160px（约4个会话项）
- 如果会话超过4个，显示滚动条
- 滚动条样式优化，与暗色主题匹配

### 3. 智能排序
1. **活跃会话优先**：当前有WebSocket连接的会话显示在最前面
2. **消息数量排序**：消息多的会话靠前
3. **名称/ID排序**：最后按字母顺序排序

### 4. 视觉指示器
- **活跃状态**：绿色圆点（●）指示有WebSocket连接的会话
- **消息计数**：右侧显示消息数量
- **当前选中**：蓝色背景高亮

## 技术实现

### 组件结构变化
```tsx
// 之前：下拉选择器
<div className={styles.sessionSelector}>
  <button onClick={() => setIsOpen(!isOpen)}>...</button>
  {isOpen && <div className={styles.sessionDropdown}>...</div>}
</div>

// 之后：固定列表
<div className={styles.sessionListContainer}>
  {sortedSessions.map(session => (
    <div className={styles.sessionListItem} onClick={handleSelect}>
      <div className={styles.sessionItemContent}>...</div>
    </div>
  ))}
</div>
```

### CSS样式变化
```css
/* 新增：固定高度容器 */
.sessionListContainer {
  max-height: 160px;
  overflow-y: auto;
  /* 自定义滚动条 */
}

/* 新增：列表项 */
.sessionListItem {
  padding: 8px 10px;
  border-bottom: 1px solid #21262d;
  cursor: pointer;
}

.sessionListItem:hover { background: #21262d; }
.sessionListItem.active { background: rgba(31, 111, 235, 0.2); }
```

## 测试验证

### 1. 视觉检查
- 启动服务：`bash scripts/start-tmux-dev.sh`
- 打开浏览器访问 `http://localhost:3000`
- 检查侧边栏会话列表是否直接显示

### 2. 功能测试
- **创建多个会话**：点击"New Session"创建会话A、B、C、D、E（超过4个）
- **检查滚动**：观察是否出现滚动条，能否滚动查看所有会话
- **切换会话**：点击不同会话，检查是否切换成功
- **活跃状态**：观察活跃会话是否有绿点指示
- **排序验证**：活跃会话是否显示在最前面

### 3. 边界情况
- **0个会话**：显示"No sessions"提示
- **1-3个会话**：容器高度自适应，不显示滚动条
- **刚好4个会话**：刚好填满容器，无滚动条
- **超过4个会话**：显示滚动条，可滚动查看

## 性能考虑

### 1. 渲染性能
- 列表项使用简单div，无复杂嵌套
- 排序计算轻量（O(n log n)，n为会话数，通常<20）
- 虚拟滚动未实现（会话数量少，无需优化）

### 2. 内存使用
- 定期轮询活跃状态（5秒间隔）
- 会话数据从store读取，无重复存储

### 3. 用户体验
- 始终可见的列表减少交互步骤
- 固定高度保持布局稳定
- 滚动条在需要时自动显示

修改遵循最小侵入原则，仅改动必要部分，保持系统稳定性。
