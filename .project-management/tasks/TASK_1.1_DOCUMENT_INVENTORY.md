## 📋 子任务分解

### 子任务 1.1.1: 列出所有文档（2小时）

**工作内容**:
1. 查找所有Markdown文件
2. 记录文件位置和大小
3. 初步统计文档数量

**命令**:
```bash
# 查找所有文档
find /root/pi-gateway-standalone -type f -name "*.md" | sort > /tmp/all_docs.txt

# 统计数量
wc -l /tmp/all_docs.txt

# 按目录分组
find /root/pi-gateway-standalone -type f -name "*.md" | xargs dirname | sort | uniq -c | sort -rn
```

**验收标准**:
- [ ] 完整的文档列表
- [ ] 按目录分类统计
- [ ] 文档大小信息

**预计时间**: 2小时  
**状态**: ⬜ 未开始

---

### 子任务 1.1.2: 读取并分类文档（6小时）

**工作内容**:
1. 读取每个文档的前50行了解内容
2. 根据内容进行分类
3. 标记文档类型

**分类标准**:
- **核心文档**: README, DEVELOPMENT, ARCHITECTURE等
- **分析报告**: 各种ANALYSIS, REPORT文档
- **修复记录**: FIX_SUMMARY, FIX_REPORT等
- **测试文档**: TEST_SUMMARY, 测试指南等
- **规范文档**: UI_LAYOUT_SPEC, ERROR_HANDLING等
- **其他**: 临时文档、个人笔记等

**工作表格**:

| 文件路径 | 文件名 | 类型 | 大小 | 最后修改 | 状态评估 |
|----------|--------|------|------|----------|----------|
| /root/pi-gateway-standalone/ | README.md | 核心 | 4KB | 2026-03-30 | ✅ 最新 |
| ... | ... | ... | ... | ... | ... |

**验收标准**:
- [ ] 所有文档分类完成
- [ ] 类型标签准确
- [ ] 内容摘要完整

**预计时间**: 6小时  
**状态**: ⬜ 未开始

---

### 子任务 1.1.3: 识别过时/重复文档（4小时）

**工作内容**:
1. 检查文档修改日期
2. 识别内容重复的文档
3. 识别已过时文档

**过时标准**:
- 最后修改超过3个月
- 内容涉及已删除的功能
- 修复报告（修复已完成）

**重复标准**:
- 内容相似度>70%
- 同一主题多个文档

**工作表格**:

| 文件 | 问题类型 | 原因 | 建议操作 |
|------|----------|------|----------|
| xxx.md | 过时 | 功能已删除 | 归档 |
| yyy.md | 重复 | 与zzz.md重复 | 合并 |

**验收标准**:
- [ ] 过时文档清单
- [ ] 重复文档清单
- [ ] 操作建议

**预计时间**: 4小时  
**状态**: ⬜ 未开始

---

### 子任务 1.1.4: 生成文档清单（4小时）

**工作内容**:
1. 整理所有收集的信息
2. 生成结构化清单文档
3. 提供重构建议

**输出格式**:
```markdown
# 文档清单

## 核心文档（保留）
- README.md - 项目说明
- ...

## 需要合并的文档
- ANALYSIS_REPORT.md + DEEP_ANALYSIS.md → ARCHITECTURE.md
- ...

## 过时可删除的文档
- FIX_SUMMARY_2026_01.md（修复已完成）
- ...

## 建议新增的文档
- CONTRIBUTING.md
- TESTING_GUIDE.md
```

**验收标准**:
- [ ] 清单文档完整
- [ ] 重构建议清晰
- [ ] 格式规范

**预计时间**: 4小时  
**状态**: ⬜ 未开始
