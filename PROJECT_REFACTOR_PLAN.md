## 🎯 项目目标

### 主要目标
1. **文档系统化** - 整合分散文档，建立清晰的文档体系
2. **代码规范化** - 添加注释、统一格式、结构化日志
3. **测试完整化** - 修正测试、补全缺失、删除废弃
4. **质量可视化** - 生成潜在Bug报告和代码质量报告

### 约束条件
- ❌ **不做逻辑改动** - 只添加注释、日志、格式化
- ❌ **不改架构** - 保持现有架构，只做表面整理
- ✅ **保持兼容性** - 所有现有功能保持正常

**优先级排序**:
1. **高优先级**（公共API）:
   - stores/*.ts - 所有store文件
   - services/api/*.ts - API服务
   - components/layout/*.tsx - 布局组件

   - lib/*.ts - 工具库
   - utils/*.ts - 工具函数

**JSDoc标准格式**:
```typescript
/**
 * @description 简要描述函数/组件功能
 * @param {Type} paramName - 参数描述
 * @returns {ReturnType} 返回值描述
 * @example
 * ```typescript
 * const result = functionName(arg1, arg2);
 * ```
 * @throws {ErrorType} 可能抛出的错误
 * @since 版本号
 */
```

#### 任务 2.2: 结构化日志改造（3天）

**当前状态**:
```
console.log数量: 244处
类型分布:
- console.log: ~150处
- console.warn: ~50处
- console.error: ~44处
```

**改造方案**:
1. **创建日志工具**
```typescript
// lib/logger.ts
export enum LogLevel {
  DEBUG = 0,
  INFO = 1,
  WARN = 2,
  ERROR = 3,
}

export interface Logger {
  debug(message: string, ...args: unknown[]): void;
  info(message: string, ...args: unknown[]): void;
  warn(message: string, ...args: unknown[]): void;
  error(message: string, ...args: unknown[]): void;
}
```

2. **日志分类**:
   - `[WebSocket]` - WebSocket相关
   - `[Store]` - 状态管理
   - `[API]` - API调用
   - `[Component]` - 组件生命周期
   - `[Session]` - 会话管理

3. **开发/生产环境区分**:
   - 开发环境：详细日志
   - 生产环境：只记录ERROR

**交付物**:
- `src/shared/lib/logger.ts`
- 244处console.log替换为结构化日志
- 日志级别配置文档
